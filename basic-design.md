## 3. ロール・権限管理

### 3.1 ロール定義

#### ロール: `user`（エンドユーザー）
- 新規登録、ログイン・ログアウト可能
- 商品検索・閲覧可能
- カート管理・注文処理可能
- 自身の注文履歴のみ閲覧可能
- 統計情報・管理機能へのアクセス不可

#### ロール: `admin`（管理者）
- 全機能へのアクセス可能
- ユーザー管理機能を利用可能
- 商品管理・在庫管理機能を利用可能
- 全注文の確認・管理可能
- 統計情報の閲覧可能
- ダッシュボードへのアクセス可能

### 3.2 アクセス制御マトリックス

| 機能 | ゲスト | ユーザー | 管理者 |
|------|--------|---------|--------|
| 会員登録 | ○ | ○ | ○ |

| 商品閲覧 | ○ | ○ | ○ |

## 7. 非機能要件定義

### 7.1 性能要件

| 項目 | 要件 | 測定方法 |
|------|------|---------|
| レスポンスタイム | 通常操作: 2秒以内、検索処理: 3秒以内 | サーバーログ、APM |
| スループット | 同時接続ユーザー数: 1,000人 | 負荷テスト |
| DBクエリ | 複雑なクエリ: 5秒以内 | クエリログ分析 |
| 画面表示速度 | First Contentful Paint: 1.5秒以内 | Lighthouse |

### 7.2 セキュリティ要件

| 項目 | 要件 | 実装方法 |
|------|------|---------|
| 認証 | JWT方式 | JWT + Refresh Token |
| パスワード | bcryptでハッシュ化(cost=10以上) | bcrypt library |
| 通信 | HTTPS必須(TLS 1.2以上) | SSL証明書導入 |
| SQLインジェクション | パラメータバインド | ORMライブラリ使用 |
| XSS | サニタイゼーション | DOMPurify |
| CSRF | トークン検証 | csurf middleware |
| 個人情報 | 暗号化保存 | AES-256 |

### 7.3 権限管理要件

| 項目 | 要件 |
|------|------|
| ロールベースアクセス制御（RBAC） | user / admin の2つのロール |
| API認可 | JWTトークンからロール情報を取得し、エンドポイント毎に検証 |
| UIアクセス制御 | ロールに応じてメニュー・ボタンの表示/非表示を制御 |
| 権限チェック | 不正なリクエストに対して401/403を返却 |
